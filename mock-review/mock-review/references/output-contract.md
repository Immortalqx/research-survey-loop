# Output Contract for Mock Review

Use this file when writing the final review artifact.

## Required Label

The final document must clearly state one of:

- `Mock Review`
- `Simulated Review`
- `Simulated Review for Author Preparation`

Do not title it as an official review.

## Recommended Top-Level Structure

If a venue-specific form is known, use that form. Otherwise use:

```markdown
# Mock Review: <Paper Title>

## Evidence and Scope

## Venue / Journal Requirements Summary

## PDF Artifact Hygiene Check

## Literature Grounding Summary

## Simulated Review

### Summary

### Strengths

### Weaknesses

### Detailed Review

### Questions Authors Should Be Ready To Answer

### Scores / Recommendation

## Rebuttal Preparation Notes

## Assumptions and Source Gaps
```

## Tone

- Write as a rigorous simulation of what a reviewer might say.
- Keep the voice professional, concrete, and evidence-based.
- Avoid official enforcement language unless quoting venue policy.
- Do not say "I am an assigned reviewer" or "as the official reviewer."
- It is acceptable to use reviewer-style first person inside the simulated-review section, but the document wrapper must make clear this is a mock review for author preparation.

## Scores

When venue-specific score scales are known, reproduce them exactly and cite the source in the evidence section.

When no score scale is known, use a generic score block:

```markdown
- Overall mock recommendation: <Reject / Weak Reject / Borderline / Weak Accept / Accept>
- Confidence: <Low / Medium / High>
- Venue fit: <Poor / Partial / Good / Strong>
- Technical quality: <Low / Medium / Good / Strong>
- Presentation: <Poor / Fair / Good / Excellent>
```

## Rebuttal Preparation Notes

This section is for authors only. Include:

- likely reviewer objections
- concise rebuttal angles
- missing experiments that would most improve the paper
- wording changes to narrow overclaims
- citation or related-work fixes

Keep these notes separate from the simulated review so the formal review text remains realistic.
